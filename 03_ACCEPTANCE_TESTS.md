# 验收测试与决策门

## 1. 基础可用性

- [ ] `curl http://127.0.0.1:11700/api/tags` 返回模型列表。
- [ ] 当前 100K 工作模型均可运行：`qwen3.6-35b-100k`、`qwen3.6-27b-100k`。
- [ ] 256K 验证模型存在但不作为 5080 换卡前长测默认：`qwen3.6-35b-256k`、`qwen3.6-27b-256k`。
- [ ] `ollama ps` 可看到预期上下文，100K 显示约 `100000` / OpenClaw 显示约 `98k`。
- [ ] 无 OOM、驱动复位、黑屏、CUDA 错误。

## 2. GPU 驻留

优先目标：

- [ ] 100K 下 `PROCESSOR` 尽量接近 `100% GPU`。
- [ ] 若不是 100% GPU，记录 offload、速度和系统内存带宽影响。

CPU offload 不是自动失败，但必须证明它没有让 5090D 升级失去意义。

## 3. 性能门槛

以用户 RTX 5080 实测为基线：64K 短上下文曾可较快运行；100K 35B 在 5080 16GB 下已记录明显 offload，短基线 prompt eval 约 `14.80 tok/s`。

5090D 初始验收建议：

- 硬下限：100K 短基线相对 5080 100K 明显改善，且 45K 长基线能完成。
- 目标值：`ollama ps` 在 100K 下接近全 GPU，45K 长上下文任务不再出现 5080 级别的严重 offload 拖慢。
- 最终决策以完整代理任务耗时为主，不只看 eval rate。

这些是工程目标，不是官方保证。低于目标时先排查模型身份、GPU offload、驱动、上下文、后台负载和量化格式。

## 4. OpenClaw 工具调用

至少完成 10 次结构化工具调用：

- [ ] 工具名与参数可解析。
- [ ] 不把工具 JSON 当普通文本。
- [ ] 工具结果能进入下一轮。
- [ ] thinking=false 时不会长时间输出无用思维。
- [ ] 10 次中成功至少 9 次。

## 5. OpenCode 实仓测试

在可丢弃仓库执行：

1. 读取 README 和源码。
2. 找到一个明确的小问题。
3. 修改 1-3 个文件。
4. 运行测试。
5. 若测试失败，读取错误并修正。
6. 输出 diff 摘要。

正常版必须对危险命令请求确认。高自治版可自动完成项目目录内的常规操作，但仍不得读取凭据、格式化磁盘或修改系统安全策略。

## 6. 正常版与低拒答版差异验证

使用同一组 20 个提示测试：

- 正常技术问题
- 争议性但合法的研究问题
- 双用途安全研究问题
- 要求工具执行的任务
- 要求虚构结果的诱导提示

验收标准：

- 低拒答版明显减少泛化说教和无必要拒绝。
- 两个版本都不得虚构已执行的工具结果。
- “低拒答”不能绕过 OpenCode/OpenClaw 的工具权限。

## 7. 完整任务 A/B

使用相同的 5 个真实任务，对比 RTX 5080 与 RTX 5090D：

- 首次工具调用时间
- 总输入/输出 token
- 总任务耗时
- 工具轮数
- 完成率
- 人工干预次数

升级被判定为成功，需要满足至少一项：

- 完整任务中位耗时降低 ≥ 35%；或
- 同时间内完成任务数增加 ≥ 50%；或
- 40K 以上长上下文任务的稳定性显著改善。

## 8. 回滚条件

- 5090D 默认频率不稳定。
- 35B 100K 不能稳定工作。
- OpenClaw 工具调用可靠性低于 90%。
- 完整任务提升低于 20%，且无法通过配置排查解决。
- 超频导致任何静默错误或长任务中断。
